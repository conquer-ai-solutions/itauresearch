The Core Constraint
Why AI Cannot Touch Data Sources Directly
Licensed data providers explicitly block direct AI agent access. This is the non-negotiable architectural constraint that shapes every design decision.
Licensing Reality
Capital IQ (S&P): Explicitly prohibits AI agents, MCP servers, and automated AI-driven systems from connecting to their APIs. The CIQ Pro API exists — but the caller must be a traditional microservice, not an AI agent. Bearer tokens expire every 60 minutes and cannot be issued to AI processes.

Bloomberg: AI/ML use of Bloomberg data requires a specific license amendment to the existing contract. Standard terminal or BLPAPI licences do not permit feeding data into AI systems. Data License (DL) with the AI/ML amendment is the compliant path.

Economatica: Commercial license with distribution restrictions. Programmatic REST API may be available but terms must be reviewed for AI-derived-output use cases.

The implication: Any architecture where an LLM, AI agent, or MCP server connects directly to these systems is non-compliant and puts Itaú at licensing risk. The architecture must enforce a hard boundary between data ingestion and AI consumption.
Solution Architecture
Decoupled Two-Layer Design
Traditional data engineering (Layer 1) handles ingestion and staging. A separate AI layer (Layer 2) reads only from the staged data lake. The red boundary between them is absolute — AI never crosses it.

Scenario B — Xpressfeed Bulk Delivery
Automated via SFTP listener
S&P pushes pipe-delimited flat files via SFTP nightly. Critical: files have no column headers — raw pipe-separated values. Parsing requires S&P's schema docs mapping column positions to field names. SFTP listener detects drops, applies schema map, handles type casting (proprietary date formats, null indicators), validates row counts against manifest. Files can be 500MB+ for universe snapshots.
S&P SFTP Push→ SFTP Listener→ Schema Mapper (no headers)→ Type Cast + Validate→ Data Lake

The Hard Problem
Private Company Research — Agentic Mosaic
Everything above assumes structured data from known sources with pre-built pipelines. Private companies break that assumption entirely. There are no Bloomberg tickers, no CVM filings, no pre-staged lake records. The system has to build the picture from scratch — in real time.
Why This Requires a Different Architecture
For public companies, data is pre-staged in the lake via scheduled ETL. For private companies, we can't do this — we don't know in advance which company the analyst will research, and no structured source covers them. The solution: an agentic orchestration layer that assembles a data mosaic in real time, with confidence scoring on every data point and source classification distinguishing verified facts from estimates.

Multi-Source Reasoning — What the AI Actually Does
Cross-Source Validation
With financials from 3–4 sources simultaneously in context, the AI can:

Detect discrepancies — Bloomberg reports Cosan EBITDA at R$8.2B but CVM filing shows R$7.9B. AI flags the difference and identifies the cause (Bloomberg includes equity method income, CVM is consolidated-only).

Pick ground truth — for regulatory metrics, CVM filing is authoritative. For consensus estimates, Bloomberg is primary. For historical LatAm data, Economatica has deeper coverage. The AI learns source hierarchy per metric type.

Fill gaps — CIQ might have M&A transaction data that Bloomberg doesn't. CONAB has crop forecasts that no financial terminal covers. By having everything in one window, nothing falls through the cracks.
Contextual Reasoning
This is what separates a 1M-token engine from a RAG lookup:

Cross-domain inference — CONAB projects sugarcane production up 8% → Raízen's ethanol margin should improve → Cosan SOTP valuation increases. The AI connects macro data to subsidiary performance to parent holding value. No human prompted this chain.

Temporal reasoning — Q3 revenue grew 12% but IBGE shows sector GDP flat. AI identifies the growth is acquisition-driven (M&A data from CIQ) not organic, and flags it in the thesis.

Internal knowledge leverage — prior pitchbook from 6 months ago had a different thesis. AI surfaces the discrepancy: "Previous analysis projected R$9B EBITDA by Q4; actual trajectory is R$7.9B. Key miss: Compass margin compression."

Enterprise & Compliance
Governance, Security & Regulatory Alignment
Itaú BBA operates under BACEN, CVM, and LGPD regulatory frameworks. This architecture is designed to satisfy compliance requirements from day one — not retrofitted later.
Data Provenance & Audit Trail
Every record tagged: source system, pull timestamp, license type, freshness, SHA256 hash. AI outputs cite sources inline. Compliance can trace any claim to the originating data point in 3 hops. Audit question: "did AI access Bloomberg directly?" → No, with full lineage proof stored in the provenance table.
Data Residency & Perimeter
Data lake, ETL services, and AI engine all deploy within Itaú's AWS VPC. No data leaves the security perimeter. Licensed data (Bloomberg, CIQ) stays within existing contract terms — processed and stored on Itaú's infrastructure, never transmitted to third-party AI services. AWS Bedrock invocations stay within the same region (sa-east-1).
LGPD Compliance
The system processes financial and market data, not personal data — LGPD exposure is minimal. Internal documents (Salesforce, CRM) may contain client names: these are processed within the VPC, never leave the perimeter, and follow Itaú's existing data classification policies. No personal data is used for model training. Audit logs record every data access.
BACEN / CVM Regulatory
AI-generated research outputs are clearly marked as AI-assisted, not autonomous. The analyst remains the author — the system provides data synthesis and drafting. This aligns with CVM Instruction 20 (research independence) and BACEN expectations on algorithmic transparency. All AI outputs include source attribution for regulatory review.
Access Control & Permissions
Role-based access: analysts see outputs, data engineers manage ETL, compliance accesses provenance and audit logs. Licensed data access restricted to users covered by Itaú's existing vendor agreements. API keys and credentials stored in AWS Secrets Manager — never in code or config files.
License Enforcement
The AI boundary isn't just architectural — it's enforced at the infrastructure layer. AI services have no network route to source system endpoints. ETL services have no route to AI endpoints. Firewall rules make license violations technically impossible, not just policy-prohibited.
Freshness & Staleness Controls
Configurable refresh per source. Bloomberg/CIQ: daily+. CVM: on filing. BCB: on COPOM decision. CONAB: monthly. Every record carries a staleness flag — AI explicitly warns analysts when data is older than its expected refresh cadence. No silent stale data.
Extensibility
New source = one new ETL adapter. Lake schema extensible via additive migrations. AI engine unchanged. Future sources (B3 direct feed, Refinitiv, alternative data providers) plug in without rearchitecting. The decoupled design means new data flows in without touching the AI layer.
